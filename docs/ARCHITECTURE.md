# Архитектура реакций на посты форума

## Схема

```
Форум (BestBB/PunBB)          Cloudflare Worker             Durable Object
       │                            │                            │
       │  GET /api/reactions        │  stub.fetch('/get')        │
       │  ?post_ids=1,2,3           │ ──────────────────────────► │
       │  &user_id=456            │                            │ чтение storage
       │ ◄───────────────────────── │ ◄────────────────────────── │
       │                            │                            │
       │  POST /api/reactions       │  stub.fetch('/set')        │
       │  {post_id, user_id, emoji} │  ?user_id=...&emoji=...   │
       │ ──────────────────────────► │ ──────────────────────────► │
       │                            │                            │ toggle + запись
       │ ◄── {counts, user_reaction}│ ◄────────────────────────── │
```

- **Durable Object** — один DO на пост: хранит `{ userId: emoji, ... }`
- **Worker** — API Gateway: батч-чтение (GET), запись с toggle (POST)
- **Виджет** — на форуме: рендер кнопок, optimistic update, нативный `fetch()`

## Проблемы и решения

### DO не принимал body через stub.fetch
- **Проблема**: `stub.fetch('http://do/set', {method:'POST', body:...})` — тело не доходило
- **Решение**: передача параметров через `URL.searchParams` + `encodeURIComponent`

### jQuery на форуме возвращал {} вместо ответа AJAX
- **Проблема**: `$.ajax` / `$.getJSON` перехватывались библиотекой форума
- **Решение**: нативный `fetch()` вместо jQuery

### POST + повторный GET к тому же DO возвращал пустоту
- **Проблема**: второй `stub.fetch` к тому же DO видел старые данные
- **Решение**: возвращаем ответ SET напрямую, без повторного GET

### FORUM.get('UserID') не работал
- **Проблема**: `FORUM.get('UserID')` искал свойство на объекте, а не глобал
- **Решение**: fallback на глобальную переменную `UserID`

### Эмодзи кодировались редактором форума
- **Проблема**: редактор превращал `👍` в `&#128077;`
- **Решение**: Unicode escape `\u{1F44D}`

### Комментарии в script были видны на странице
- **Проблема**: редактор форума выводил текст вне `<script>` как обычный текст
- **Решение**: удалить все комментарии из инжектируемого скрипта

### Worker без защиты Origin
- **Проблема**: любой сайт мог делать POST-запросы к Worker, израсходовав лимит
- **Решение**: белый список доменов через `ALLOWED_ORIGINS` (env var), 403 для остальных
